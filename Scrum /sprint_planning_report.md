# Sprint Planning Report – Short Guideline

### 1. Create the report
- Use a simple format (Markdown or PDF).
- Keep it short: **1–2 pages max**.
- Include the following sections:
  - **Sprint Number & Dates**
  - **Sprint Goal**
  - **Selected Product Backlog Items**
  - **Planned Tasks / Breakdown**
  - **Team Capacity & Assumptions**
  - **Definition of Done**

### 2. Name the file
- Example: `Sprint_x_Planning_Report.md` or `Sprint_x_Planning_Report.pdf`

### 3. Upload to GitHub
- Open your **project repository** on GitHub.
- Navigate to the appropriate folder (e.g. `/docs/sprint_report/`).
- Click **Add file → Upload files**.
- Upload the report and **commit** with a clear message  
  (e.g. *“Add Sprint 1 planning report”*).

### 4. Verify
- Make sure the file is visible in the repository.
